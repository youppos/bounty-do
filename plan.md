# 游戏化待办/备忘录 APP - MVP 规划文档

## 1. 产品愿景与核心定位
一款主打**“游戏化激励”**与**“高颜值自定义”**的跨平台移动端（iOS/Android）待办与备忘录应用。
通过完成任务获取金币、炫酷的视觉音效反馈、以及个性化的正/反向语音激励，打破传统待办工具的枯燥感，让目标管理像打游戏一样令人上瘾。

---

## 2. 核心功能规划 (MVP阶段)

### 2.1 任务与备忘录管理
* **基础功能**：待办事项与备忘录的增删改查。
* **金币权重系统**：
  * 用户可按任务类别或自定义重要程度，设定完成该任务可获得的任务金币数。
  * *备注：二期将引入 AI/RAG 根据任务描述自动评估价值。*
* **布局与展示**：
  * **列表模式**：默认采用长宽比为 **3:1 的卡片**列表展示。
  * **宫格模式**：支持一键切换为宫格视图。
  * **权重色彩**：卡片颜色根据任务权重/重要程度区分，内置一套炫酷的系统推荐配色，同时支持用户完全自定义颜色。

### 2.2 游戏化与激励系统
* **正向反馈（完成任务）**：
  * 触发专属的“金币进账” UI 特效与音效。
* **负向/趣味反馈（未完成/逾期任务）**：
  * 引入类似“高德导航语音包”的提示机制。
  * **安慰模式**：温和、鼓励的提示语。
  * **激进模式**：硬核、毒舌甚至“骂人”风格的催促提示。
  * 支持用户自定义这些提示语或声音机制。

### 2.3 UI/UX 设计规范 (前端重点)
* **设计语言**：致敬 Apple 苹果风，以**液态玻璃效果 (Glassmorphism)** 为默认基调，强调高级感、毛玻璃毛边、光影与流畅的微动画。
* **主题系统**：默认提供 **5 种** 高质量预设主题风，支持自由切换。
* **字体系统**：默认采用类似苹果设计规范的无衬线字体（如 San Francisco 或 Inter），同时支持用户切换自定义字体。

## 3. 技术架构与扩展性设计

为了保证前期前端快速开发，且方便后期接入后端及 AI 能力，整体架构需遵循以下原则：

* **技术栈**：Flutter (Dart) - 确保完美跨 iOS 和 Android。
* **状态管理**：推荐使用 **Riverpod** 或 **GetX**（按需选择），将 UI 与业务逻辑彻底分离。
* **数据层 (Repository 模式)**：
  * 前期：实现 `LocalTaskRepository`，使用 SQLite 或 Hive 进行本地数据持久化，配合 Mock 数据模拟网络请求。
  * 后期：预留好 `ITaskRepository` 接口，待后端就绪后，只需新增 `RemoteTaskRepository` 实现接口对接即可，UI 层无需任何修改。
* **预留接口定义**：
  * `createTask(Task task)`
  * `completeTask(String taskId, TaskCompletionConfig config)`
  * `getAiTaskValuation(String taskDescription)` (二期)
